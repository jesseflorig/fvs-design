# Implementation Plan: Fix All Accessibility Violations

**Branch**: `002-fix-a11y-violations` | **Date**: 2026-04-19 | **Spec**: [spec.md](./spec.md)

## Summary

Resolve all WCAG 2.1 AA accessibility violations across the six existing components (Button, Badge, Card, Input, DataTable, Divider) and both themes. Violations fall into three categories: missing label associations in form controls, suppressed/incorrect focus styles in Button, and insufficient contrast on `--fg-subtle` text at micro sizes. No new components or APIs are introduced — every change is a targeted correction to existing markup, ARIA attributes, and one token value.

## Implementation Approach

### Fix 1 — Button: Transparent outline base (replaces `outline: 'none'`)

In `Button.tsx`, change the base style:
```
outline: '2px solid transparent',
outlineOffset: '2px',
```
In `onBlur`, reset to `'2px solid transparent'` (not `'none'`). This eliminates the window where the browser default ring could appear or the ring could be entirely absent.

### Fix 2 — Badge: aria-hidden on dot

Add `aria-hidden="true"` to the dot `<span>` in Badge.tsx. One-line change.

### Fix 3 — DataTable: scope="col" on headers

Add `scope="col"` to each `<th>` in the thead map. One-line addition.

### Fix 4 — Input/Select: label association via useId

```tsx
const id = useId();
// <label htmlFor={id}>
// <input id={id} ... />
```

Repeat for Select. No prop API change.

### Fix 5 — Input/Select: hint/error via aria-describedby

```tsx
const hintId = useId();
// <input aria-describedby={hint || error ? hintId : undefined} ... />
// <span id={hintId}>{error ?? hint}</span>
```

### Fix 6 — Toggle: aria-labelledby

```tsx
const labelId = useId();
// <span id={labelId} style={labelStyle}>{label}</span>
// <button aria-labelledby={labelId} role="switch" ...>
```

### Fix 7 — Checkbox: aria-labelledby

Same pattern as Toggle for the label `<span>` and `<button role="checkbox">`.

### Fix 8 — tokens.css: --fg-subtle contrast

Compute exact values to achieve ≥4.5:1:
- Light theme: `--fvs-steel` (#8A8A93) → target ≥ #6E6E76 on #FBFAF7. Exact value TBD at implementation.
- Console theme: `--fg-subtle` override in `[data-theme="console"]` → target ≥ #8A8A93 on #151517. Exact value TBD.

Token change is isolated to `tokens.css`. Both role tokens (`--fg-subtle`) and any direct usage must be re-validated after the change.
